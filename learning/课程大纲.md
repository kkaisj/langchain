# LangChain 学习课程大纲

> **核心理念**：先用起来建立整体认知，再逐步拆解深入。不求一次学完所有细节。

---

## 课程总览：12 节课

```
第 1–3 课   快速上手，建立感性认识（build first）
第 4–6 课   拆解 Agent 三大要素：模型、工具、Prompt
第 7–9 课   进阶能力：中间件、RAG、结构化输出
第 10–12 课 扩展视野：多 Agent、LangGraph 入门、工程化
```

---

## 第 1 课：Hello World —— 5 分钟创建你的第一个 Agent

**目标**：不纠结原理，先跑起来，看到效果。

- `create_agent()` 最小示例
- 给 Agent 一个工具（如获取天气）
- `agent.invoke()` 看结果

**参考文档**：
- [Quickstart](https://docs.langchain.com/oss/python/langchain/quickstart) — 官方快速入门
- [LangChain overview](https://docs.langchain.com/oss/python/langchain/overview) — 框架定位

**动手**：用你自己的 API Key 跑通第一个 agent，改改 system_prompt 看行为变化。

---

## 第 2 课：Agent = Model + Harness —— 理解核心公式

**目标**：建立 LangChain 的核心心智模型。

- 什么是 Agent？本质是"模型在循环中调用工具直到完成任务"
- `create_agent()` 的三个核心参数：`model`、`tools`、`system_prompt`
- 概览 LangChain 生态位：Deep Agents（开箱即用） → LangChain（可定制） → LangGraph（底层编排）

**参考文档**：
- [Philosophy](https://docs.langchain.com/oss/python/langchain/philosophy) — 了解框架演进的"为什么"
- [Frameworks, runtimes, and harnesses](https://docs.langchain.com/oss/python/concepts/products) — 三个层级的关系

**动手**：画出 LangChain 生态的三个层级，用自己的话解释什么时候该用哪个。

---

## 第 3 课：Agent 的输入输出 —— 消息系统

**目标**：理解 Agent 和人、Agent 和工具之间怎么"对话"。

- `HumanMessage`、`AIMessage`、`ToolMessage`、`SystemMessage`
- `agent.invoke({"messages": [...]})` 的输入格式
- 响应中的 `content_blocks` 和 `tool_calls`
- 消息是不可变的（immutable）

**参考文档**：
- [Messages](https://docs.langchain.com/oss/python/langchain/messages) — 消息类型详解

**深入源码（可选）**：
- `libs/core/langchain_core/messages/` — 看 `human.py`、`ai.py`、`tool.py`

---

## 第 4 课：模型（Model）—— Agent 的大脑

**目标**：理解如何选择和配置模型。

- 模型字符串格式：`"provider:model_name"`（如 `"openai:gpt-4o"`）
- 支持的提供商：OpenAI、Anthropic、Google、Ollama 等
- 基础参数：`temperature`、`max_tokens`
- `model.bind_tools()` — 模型如何"知道"有哪些工具可用

**参考文档**：
- [Models](https://docs.langchain.com/oss/python/langchain/models) — 模型配置详解
- [Providers overview](https://docs.langchain.com/oss/python/integrations/providers/overview) — 所有支持的模型提供商

**动手**：换一个模型提供商（如从 OpenAI 换成 Anthropic），观察行为差异。

---

## 第 5 课：工具（Tools）—— Agent 的手脚

**目标**：学会定义和配置工具。

- `@tool` 装饰器：把普通函数变成工具
- `StructuredTool`：定义复杂输入 schema
- docstring 的重要性：它是工具的"使用说明书"（LLM 通过它理解工具）
- 工具的错误处理：返回错误信息而不是抛异常

**参考文档**：
- [Tools](https://docs.langchain.com/oss/python/langchain/tools) — 工具定义与使用

**深入源码（可选）**：
- `libs/core/langchain_core/tools/simple.py` — `@tool` 装饰器的实现
- `libs/core/langchain_core/tools/render.py` — 工具如何被"翻译"给 LLM

---

## 第 6 课：System Prompt —— Agent 的性格

**目标**：学会通过 prompt 控制 Agent 的行为和风格。

- `system_prompt` 的作用：设定角色、约束行为、指定输出格式
- Prompt 中的变量：动态注入上下文信息
- 好的 system_prompt 怎么写（角色 + 边界 + 格式要求）

**参考文档**：
- [Agents](https://docs.langchain.com/oss/python/langchain/agents) — system_prompt 相关章节

**动手**：同一个工具，写 3 种不同风格的 system_prompt，观察 Agent 回答的差异。

---

## 第 7 课：中间件（Middleware）—— 控制 Agent 的行为

**目标**：理解中间件模式，学会在 Agent 循环中插入自定义逻辑。

- 什么是中间件？在 Agent 循环的不同阶段插入钩子
- 内置中间件概览（工具调用限制、人机协作、摘要压缩等）
- 编写自定义中间件

**参考文档**：
- [Middleware overview](https://docs.langchain.com/oss/python/langchain/middleware/overview)
- [Built-in middleware](https://docs.langchain.com/oss/python/langchain/middleware/built-in)
- [Custom middleware](https://docs.langchain.com/oss/python/langchain/middleware/custom)

**动手**：写一个最简单的中间件，在每次工具调用前后打印日志。

---

## 第 8 课：RAG —— 让 Agent 拥有"知识库"

**目标**：理解检索增强生成（RAG）在 LangChain 中的实现。

- 文档加载 → 分割 → 向量化 → 检索 → 生成
- 把检索器包装成 Tool
- 对比：有 RAG 的 Agent vs 纯 LLM Agent

**参考文档**：
- [RAG](https://docs.langchain.com/oss/python/langchain/rag) — RAG Agent 教程
- [Semantic Search tutorial](https://docs.langchain.com/oss/python/learn) — 从 PDF 构建搜索引擎

**深入源码（可选）**：
- `libs/core/langchain_core/vectorstores/` — `VectorStore` 抽象
- `libs/core/langchain_core/tools/retriever.py` — 检索器如何变成 Tool

---

## 第 9 课：结构化输出 —— 让 Agent 返回可编程的数据

**目标**：让 Agent 的输出不是自由文本，而是结构化的 JSON/Pydantic 对象。

- `with_structured_output()` — 约束 Agent 输出格式
- 用 Pydantic 模型定义输出 schema
- 结构化输出 vs 自由文本的使用场景

**参考文档**：
- [Structured output](https://docs.langchain.com/oss/python/langchain/structured-output) — 结构化输出指南

**动手**：让 Agent 分析一段文本并返回 `{sentiment, keywords, summary}` 结构。

---

## 第 10 课：多 Agent 模式 —— 分工协作

**目标**：理解多 Agent 系统的设计模式。

- Sub-agents：主 Agent 将子任务委托给专业子 Agent
- Handoffs：Agent 之间交接控制权
- Router：根据请求类型路由到不同的专业 Agent
- Skills：按需加载专业能力

**参考文档**：
- [Multi-agent overview](https://docs.langchain.com/oss/python/langchain/multi-agent) — 多 Agent 模式总览
- 具体模式教程：Subagents、Handoffs、Router、Skills

**动手**：创建一个主 Agent + 2 个子 Agent（一个负责翻译，一个负责总结）。

---

## 第 11 课：LangGraph 入门 —— 打开底层控制的大门

**目标**：了解 LangGraph 是什么，什么时候需要用。

- LCEL vs LangGraph：什么时候从 Chain 升级到 Graph
- StateGraph 基础概念：节点、边、状态
- 一个简单的 LangGraph 示例

**参考文档**：
- [LangGraph overview](https://docs.langchain.com/oss/python/langgraph/overview)
- [LangGraph quickstart](https://docs.langchain.com/oss/python/langgraph/quickstart)

**注意**：这只是一瞥，LangGraph 本身值得一个单独的课程。重点是理解"什么场景下需要 LangGraph"。

---

## 第 12 课：测试、调试与部署

**目标**：了解如何把 Agent 从实验推向生产。

- 单元测试：如何测试 Agent 和 Tool
- LangSmith 追踪：可视化 Agent 的每一次决策
- 评估（Evals）：怎么知道你的 Agent 表现好不好
- 部署选项概览（LangServe、LangGraph Cloud）

**参考文档**：
- [Testing overview](https://docs.langchain.com/oss/python/langchain/test)
- [LangSmith observability](https://docs.langchain.com/oss/python/langchain/observability)

---

## 附录

### 学习路径图

```
第 1–3 课         第 4–6 课          第 7–9 课          第 10–12 课
快速上手 ──────► 逐个拆解 ──────► 进阶能力 ──────► 扩展视野

"跑起来"         "会配置"           "能控制"           "懂生态"
```

### 每节课的推荐学习方式

1. **读文档**（15 分钟）— 先看本节指定的官方文档
2. **动手写**（30 分钟）— 在 `learning/demos/` 下新建 `.py` 文件练习
3. **读源码**（可选，15 分钟）— 带着问题看源码，不是"为了看而看"

### 什么时候读源码？

- 当你好奇"这个是怎么做到的"的时候
- 当你遇到不符合预期的行为，想弄清楚原因的时候
- 当你想贡献代码，需要理解内部实现的时候

**不建议**一开始就逐行读源码，容易迷失在细节里。
