# LangChain 源码学习指南

> 基于 langchain-ai/langchain 官方仓库，2026-06 版本

---

## 一、仓库整体架构

```
langchain-master/
└── libs/
    ├── core/                  # langchain-core：所有抽象基类与协议定义
    ├── langchain_v1/          # langchain（当前活跃主包）：Agent、ChatModel、Tool 等高层封装
    ├── langchain/             # langchain-classic（遗留包，不再新增功能）
    ├── partners/              # 第三方集成：OpenAI、Anthropic、Ollama 等
    │   ├── openai/
    │   ├── anthropic/
    │   ├── ollama/
    │   ├── deepseek/
    │   └── ... (共 15 个)
    ├── text-splitters/        # 文档分块工具
    ├── standard-tests/        # 集成测试共享套件
    └── model-profiles/        # 模型配置与 profile 生成
```

**分层架构**（由底向上）：

```
partners/ (集成层) → langchain_v1/ (实现层) → core/ (抽象层)
```

## 二、推荐学习路线（5 个阶段）

### 阶段 1：理解核心抽象（core 包）

**目标**：理解 LangChain 的设计哲学 —— 一切皆 `Runnable`

**关键文件**：

| 文件 | 学习重点 |
|------|---------|
| `libs/core/langchain_core/runnables/base.py` | `Runnable` 基类，`invoke`/`stream`/`batch` 等核心接口 |
| `libs/core/langchain_core/runnables/branch.py` | `RunnableBranch`，条件路由 |
| `libs/core/langchain_core/runnables/configurable.py` | 运行时配置切换 |
| `libs/core/langchain_core/runnables/fallbacks.py` | 降级容错机制 |
| `libs/core/langchain_core/runnables/passthrough.py` | `RunnablePassthrough`，数据透传 |
| `libs/core/langchain_core/language_models/base.py` | `BaseLanguageModel` 抽象 |
| `libs/core/langchain_core/language_models/chat_models.py` | `BaseChatModel` 抽象（重点！） |
| `libs/core/langchain_core/messages/` | 消息系统（HumanMessage, AIMessage, ToolMessage 等） |
| `libs/core/langchain_core/tools/` | Tool 抽象定义 |
| `libs/core/langchain_core/prompts/` | Prompt 模板系统 |
| `libs/core/langchain_core/output_parsers/` | 输出解析器 |
| `libs/core/langchain_core/callbacks/` | 回调事件系统 |

**学习技巧**：
- 先用 `uv run --group test pytest tests/unit_tests/ -k "runnable" -v` 跑相关测试理解用法
- 重点关注 `Runnable` 接口：`invoke`、`ainvoke`、`stream`、`astream`、`batch`、`pipe`（`|` 运算符）

---

### 阶段 2：理解 Chain 的组装方式（LCEL）

**目标**：掌握 LangChain Expression Language（LCEL），理解链式调用的内部机制

**关键文件**：

| 文件 | 学习重点 |
|------|---------|
| `libs/core/langchain_core/runnables/base.py` | `RunnableSequence`（`|` 链接的底层实现） |
| `libs/core/langchain_core/runnables/router.py` | `RunnableRouter`，动态路由 |
| `libs/core/langchain_core/runnables/retry.py` | 重试策略 |
| `libs/core/langchain_core/runnables/graph.py` | Chain 的可视化（ASCII / Mermaid） |
| `libs/core/langchain_core/runnables/schema.py` | StreamEvent 等事件 schema |

**动手实验**：

```python
from langchain_core.runnables import RunnableLambda

# 亲手构建一个小链，理解数据如何在 Runnable 间流动
step1 = RunnableLambda(lambda x: x + 1)
step2 = RunnableLambda(lambda x: x * 2)
chain = step1 | step2
print(chain.invoke(3))  # (3+1)*2 = 8
```

### 阶段 3：深入 Agent 系统（langchain_v1 包）

**目标**：理解现代 Agent 的工作原理

**关键文件**：

| 文件 | 学习重点 |
|------|---------|
| `libs/langchain_v1/langchain/agents/` | Agent 实现（`create_agent` 入口） |
| `libs/langchain_v1/langchain/chat_models/` | 各类 ChatModel 封装 |
| `libs/langchain_v1/langchain/tools/` | 工具集成 |
| `libs/core/langchain_core/agents.py` | AgentAction / AgentFinish schema（兼容层） |

**核心流程**：

```
用户输入 → Agent → LLM 推理 → Tool 调用 → 观察结果 → LLM 推理 → ... → 最终输出
         └─────────── 循环直到 Agent 决定停止 ───────────┘
```

## 三、阅读源码的技巧

### 3.1 从测试入手

测试是最好的使用文档。每个模块都有对应的 `tests/unit_tests/`：

```bash
# 设置依赖
cd libs/core && uv sync --all-groups

# 跑单个测试文件
uv run --group test pytest tests/unit_tests/runnables/test_runnable.py -v

# 跑特定测试函数
uv run --group test pytest tests/unit_tests/ -k "test_invoke" -v
```

### 3.2 善用文档 MCP

本仓库配置了 `.mcp.json`，可直接查询官方文档：

```
# 在对话中即可调用，无需额外配置
"Search langchain docs: how to create an agent"
```

### 3.3 阅读顺序建议

对于每个模块，按以下顺序阅读：
1. 模块的 `__init__.py` — 了解导出了哪些公共 API
2. 接口/抽象类（如 `base.py`）— 理解设计契约
3. 具体实现（如 `chat_models.py`）— 看实现细节
4. 对应的测试文件 — 理解使用方式和边界条件

### 3.4 动手写 Demo

新建 `learning/demos/` 目录，放自己的实验代码：

```python
# learning/demos/01_hello_runnable.py
import os
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

os.environ["OPENAI_API_KEY"] = "your-key"

llm = ChatOpenAI(model="gpt-4o")
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有帮助的助手"),
    ("user", "{input}")
])
chain = prompt | llm

result = chain.invoke({"input": "你好"})
print(result.content)
```

### 3.5 画架构图

学完一个模块后，用 Mermaid 画出类图和调用流程：

```bash
# langchain 支持将 chain 导出为 Mermaid
chain.get_graph().draw_mermaid()
```

## 四、常见陷阱与注意事项

1. **不要改 core 的公共接口** — core 包的 API 保持向后兼容，随意修改会导致生态崩溃
2. **区分 langchain (classic) 和 langchain_v1** — `libs/langchain/` 是遗留代码，不再新增功能。新的特性在 `libs/langchain_v1/`
3. **类型标注是必须的** — 所有代码都有完整类型标注，读代码时关注类型能快速理解数据流
4. **Runnable 是万物的基础** — 如果你不理解某个组件，先看它是否继承自 `Runnable`，再看它实现了哪些方法
5. **消息是不可变的** — `BaseMessage` 及其子类是不可变对象，修改会返回新实例
